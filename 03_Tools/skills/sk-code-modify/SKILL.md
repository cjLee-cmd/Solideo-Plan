**[필수] 출력 형식 규칙: 모든 결과는 반드시 마크다운(.md) 파일로 저장한다. .json 파일을 생성하지 마라. 절대 .json 확장자를 사용하지 마라.**


수정 대상 코드를 읽고 요구사항에 따라 수정하라.

## 수정 규칙

- **최소 변경 원칙**: 요구사항에 필요한 최소 범위만 수정하라. 불필요한 리팩토링을 동시에 하지 마라.
- **스타일 유지**: 기존 코드 스타일을 유지하라.
- **변경 추적**: 수정한 모든 파일의 변경 전/후를 기록하라. 변경 이유를 주석으로 남겨라.
- **하위 호환성**: 기존 API 시그니처를 변경하지 마라 (불가피한 경우 사유 명시). 기존 데이터 형식과의 호환성을 유지하라.
- **DB 변경**: 스키마 변경 시 마이그레이션 스크립트를 포함하라.
- **보안 유지**: 수정으로 인해 기존 보안 로직이 우회되지 않도록 하라. 새로 추가되는 입력은 검증 로직을 포함하라.

## 출력

- 수정된 소스코드
- `90_Result_Doc/change_summary.md`에 변경 파일 목록, diff 요약, 수정 사유를 기록하라
