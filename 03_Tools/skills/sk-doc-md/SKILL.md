**[필수] 출력 형식 규칙: 모든 결과는 반드시 마크다운(.md) 파일로 저장한다. .json 파일을 생성하지 마라. 절대 .json 확장자를 사용하지 마라.**


문서 유형과 원본 데이터를 확인하고, 해당 유형에 맞는 마크다운 문서를 작성하라.

## 문서 유형별 구성

### 설계서(design)
문서 정보, 프로젝트 개요, 아키텍처, 모듈 구조, API, DB 스키마, 기술스택, 보안, 변경 이력

### 분석서(analysis)
문서 정보, 분석 대상 개요, 기술스택, 구조, 의존성, 코드 메트릭스, 개선 제안

### 테스트 보고서(test-report)
문서 정보, 테스트 개요, 환경, 결과 요약, 상세 결과, 실패 분석, 커버리지, 결론

### 회의록(minutes)
회의 정보(일시, 장소, 참석자), 안건, 논의 내용, 결정 사항, 후속 조치(담당자, 기한)

### 제안서(proposal)
문서 정보, 제안 배경, 현황 분석, 제안 내용, 기대 효과, 추진 일정, 소요 자원

### 매뉴얼(manual)
문서 정보, 개요, 설치/설정, 사용 방법, 주의사항, FAQ

## 작성 규칙

- 한국어로 작성하라
- 표와 목록을 적극 활용하라
- 기술 용어에 간단한 설명을 병기하라

## 저장 경로

생성된 문서를 `90_Result_Doc/` 디렉토리에 저장하라. 파일명은 문서 유형에 따라 결정하라.
