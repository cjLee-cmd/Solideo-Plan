**[필수] 출력 형식 규칙: 모든 결과는 반드시 마크다운(.md) 파일로 저장한다. .json 파일을 생성하지 마라. 절대 .json 확장자를 사용하지 마라.**


file_manifest.md와 테스트 결과를 읽고 종합 보고서를 작성하라.

## 보고서 구성

1. **생성 요약** - 생성일, 설계 문서 기준, 생성 범위, 총 파일 수/코드 라인 수
2. **생성 파일 목록** - 표(파일 경로, 유형, 라인 수, 설명)
3. **설계 대비 구현 현황** - design.md 대비 구현 완료/미완료 항목
4. **테스트 결과** - 요약(총/통과/실패/스킵, 커버리지), 실패 테스트 상세
5. **미해결 사항** - 미구현 항목, 추가 작업, 알려진 제한사항
6. **다음 단계** - 권장 후속 작업

`90_Result_Doc/generation_report.md`로 저장하라.
