**[필수] 출력 형식 규칙: 모든 결과는 반드시 마크다운(.md) 파일로 저장한다. .json 파일을 생성하지 마라. 절대 .json 확장자를 사용하지 마라.**


change_summary.md와 프로젝트 코드를 읽고 영향 범위를 분석하라.

## 분석 항목

1. **직접 영향** - 변경된 함수/메서드를 호출하는 코드, 변경된 클래스를 상속/구현하는 코드, 변경된 인터페이스에 의존하는 코드
2. **간접 영향** - 변경된 모듈에 의존하는 다른 모듈, 변경된 데이터 구조를 사용하는 코드, 설정 변경의 파급 효과
3. **DB 영향** - 스키마 변경 시 영향받는 쿼리/매퍼, 데이터 마이그레이션 필요 여부
4. **API 영향** - 시그니처 변경 시 클라이언트 영향, 응답 형식 변경 시 영향

## 위험도 평가

- **높음**: 핵심 기능에 영향, 장애 가능 → 사용자 확인 필수
- **중간**: 일부 기능에 영향 → 관련 테스트 실행 필요
- **낮음**: 경미한 영향 → 일반 테스트로 충분

## 출력

영향 분석 결과를 표(영향 유형, 파일/모듈, 위험도, 설명, 조치 필요 여부)로 정리하여 `90_Result_Doc/impact_report.md`에 저장하라.
