**[필수] 출력 형식 규칙: 모든 결과는 반드시 마크다운(.md) 파일로 저장한다. .json 파일을 생성하지 마라. 절대 .json 확장자를 사용하지 마라.**


design.md를 읽고 설계에 따라 소스코드를 생성하라.

## 생성 규칙

- design.md의 디렉토리 구조를 그대로 따르라
- 계층별 패키지/모듈 분리를 준수하라
- 설정 파일(application.yml, .env 등)을 포함하라
- 함수/메서드는 단일 책임 원칙을 따르라
- 변수명/함수명은 설계 문서의 용어를 사용하라
- 주요 로직에 한국어 주석을 포함하라
- 에러 처리를 포함하라

## 보안 필수 사항

- 입력값 검증을 포함하라
- SQL 파라미터 바인딩을 사용하라
- 인증/인가 코드를 설계에 따라 구현하라
- 감사 로그 코드를 포함하라

## 폐쇄망 규칙

- 외부 네트워크 호출 코드를 포함하지 마라
- 모든 의존성은 내부 저장소에서 제공 가능한 것만 사용하라
- CDN 참조 대신 로컬 리소스를 사용하라

## 출력

- 생성된 소스코드 파일들 (디렉토리 구조 포함)
- `90_Result_Doc/file_manifest.md`에 생성 파일 목록과 설명을 기록하라
