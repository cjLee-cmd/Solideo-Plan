**[필수] 출력 형식 규칙: 모든 결과는 반드시 마크다운(.md) 파일로 저장한다. JSON(.json) 파일을 생성하지 마라. 절대 .json 확장자를 사용하지 마라.**

지정된 프로젝트 경로의 소스코드를 분석하여 아래 항목을 도출하라.

## 분석 항목

1. **기술스택 식별** - 프로그래밍 언어(버전), 프레임워크/라이브러리, 빌드 도구, 데이터베이스, 미들웨어
2. **디렉토리 구조 매핑** - 최상위 디렉토리 트리, 주요 디렉토리별 역할, 설정 파일 목록/용도
3. **모듈 의존성 분석** - 내부 모듈 간 의존 관계, 외부 라이브러리 의존성, 순환 의존성 여부
4. **코드 메트릭스** - 총 파일 수/코드 라인 수, 언어별 비율, 복잡도 높은 파일 식별
5. **폐쇄망 호환성 점검** - 외부 네트워크 호출 코드 식별, 외부 CDN/API 의존 여부, 라이선스 확인 필요 항목

## 분석 규칙

- 빌드 설정 파일(pom.xml, build.gradle, package.json, .csproj 등)을 먼저 확인하라
- 바이너리 파일과 빌드 산출물 디렉토리는 제외하라
- 외부 의존성은 폐쇄망 호환 여부를 반드시 표시하라
- 분석 결과는 객관적 근거(파일 경로, 코드 라인)와 함께 제시하라

## 출력

**반드시 `90_Result_Doc/analysis_result.md`에 마크다운 형식으로 저장하라.**
- 표와 목록으로 구조화하라
- .json 파일을 생성하지 마라
- 90_Result_Doc 디렉토리가 없으면 먼저 생성하라
