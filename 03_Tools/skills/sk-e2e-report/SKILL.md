**[필수] 출력 형식 규칙: 모든 결과는 반드시 마크다운(.md) 파일로 저장한다. .json 파일을 생성하지 마라. 절대 .json 확장자를 사용하지 마라.**

E2E 테스트 매니페스트와 실행 결과를 읽고 종합 보고서를 작성하라.

## 보고서 구성

1. **테스트 요약** - 실행일, 대상 시스템, 총 시나리오/통과/실패/스킵
2. **시나리오 목록** - 표(시나리오명, 유형(웹UI/API/비즈니스), 결과, 실행시간)
3. **웹 UI 테스트 결과** - Playwright 테스트 상세 (브라우저, 페이지 흐름, 스크린샷)
4. **API 통합 테스트 결과** - 엔드포인트 연계 흐름, 응답 검증
5. **실패 분석** - 실패 시나리오별 원인, 재현 단계, 스크린샷 경로
6. **배포 판정** - Critical 시나리오 전체 통과 여부 기준 배포 가능/불가
7. **개선 제안** - 추가 시나리오 필요 영역, 불안정 테스트 식별

## 출력

**반드시 `90_Result_Doc/e2e_report.md`에 마크다운 형식으로 저장하라.**
- 90_Result_Doc 디렉토리가 없으면 먼저 생성하라
