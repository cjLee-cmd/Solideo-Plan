# SK_impact_check 사용자 매뉴얼

## 개요

코드 수정 후 영향받는 코드 영역을 검토하는 Skill이다.

## opencode에서 사용하기

### 기본 호출

```
/skill SK_impact_check --changes change_summary.md --project_path ./
```

## 분석 결과 항목

| 항목 | 설명 |
|------|------|
| 직접 영향 | 변경 코드를 호출/참조하는 파일 |
| 간접 영향 | 변경 모듈에 의존하는 다른 모듈 |
| DB 영향 | 스키마 변경 시 관련 쿼리/매퍼 |
| API 영향 | 시그니처/응답 변경의 클라이언트 영향 |
| 위험도 | 높음/중간/낮음 등급 |

## 위험도 등급

| 등급 | 의미 | 조치 |
|------|------|------|
| 높음 | 핵심 기능에 영향, 장애 가능 | 사용자 확인 필수, 추가 테스트 |
| 중간 | 일부 기능에 영향 가능 | 관련 테스트 실행 |
| 낮음 | 경미한 영향 | 일반 테스트로 충분 |

## 후속 작업

- 고위험 항목이 있으면 사용자 확인 후 다음 단계 진행
- SK_test_gen으로 영향받는 영역의 추가 테스트 생성

## 주의사항

- 고위험 항목이 발견되면 SW_modification Workflow가 자동 중지된다
- SSA_impact_analyzer가 심층 분석을 보조한다
