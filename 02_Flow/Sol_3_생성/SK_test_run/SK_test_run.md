# SK_test_run

## 목적

생성된 테스트 코드를 실행하고, 결과(통과/실패, 커버리지)를 수집한다. SM_test_runner MCP를 통해 실제 테스트 프레임워크를 실행한다.

## 입력 파라미터

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| test_path | Y | 테스트 코드 디렉토리 경로 |
| config | N | 테스트 실행 설정 파일 경로 |
| coverage | N | 커버리지 측정 여부 (true/false). 기본: true |

## 실행 프롬프트

```
당신은 테스트 실행 관리자입니다.
SM_test_runner MCP를 사용하여 테스트를 실행하고 결과를 수집하십시오.

[테스트 경로]
{test_path}

[실행 절차]
1. SM_test_runner의 run_tests 도구로 테스트 실행
2. 커버리지 측정이 활성화된 경우 get_coverage 도구로 커버리지 수집
3. 실패한 테스트가 있으면 get_test_detail로 상세 정보 수집

[결과 수집]
- 총 테스트 수
- 통과/실패/스킵 수
- 실패 테스트 목록 (테스트명, 실패 원인, 스택트레이스)
- 커버리지 비율 (라인/브랜치)
- 실행 시간

[판단]
- 모든 테스트 통과 + 목표 커버리지 달성: 성공
- 테스트 실패 있음: 실패 (상세 원인 보고)
- 커버리지 미달: 경고 (추가 테스트 필요)
```

## 출력

| 산출물 | 형식 | 설명 |
|--------|------|------|
| test_result | JSON | 테스트 실행 결과 구조체 |

## MCP 연동

SM_test_runner의 다음 도구를 사용한다:
- `run_tests`: 테스트 실행
- `get_coverage`: 커버리지 측정
- `get_test_detail`: 실패 테스트 상세
