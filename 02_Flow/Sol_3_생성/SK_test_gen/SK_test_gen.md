# SK_test_gen

## 목적

생성 또는 수정된 소스코드에 대해 단위 테스트와 통합 테스트 코드를 자동 생성한다.

## 입력 파라미터

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| source_path | Y | 테스트 대상 소스코드 경로 |
| test_framework | N | 테스트 프레임워크 (auto/junit/pytest/jest/nunit). 기본: auto |
| coverage_target | N | 목표 커버리지 (%). 기본: 80 |
| test_type | N | 테스트 유형 (unit/integration/both). 기본: both |

## 실행 프롬프트

```
당신은 소프트웨어 테스트 전문가입니다.
아래 소스코드에 대한 테스트 코드를 생성하십시오.

[대상 소스코드]
경로: {source_path}

[테스트 설정]
프레임워크: {test_framework}
목표 커버리지: {coverage_target}%
테스트 유형: {test_type}

[테스트 코드 생성 규칙]

1. 단위 테스트
   - 각 public 메서드/함수에 대해 최소 1개 이상의 테스트 작성
   - 정상 케이스, 경계값, 예외 케이스를 포함
   - 테스트명은 한국어로 의미를 명확히 기술
   - Given-When-Then 패턴 사용

2. 통합 테스트
   - 주요 API 엔드포인트에 대해 요청/응답 테스트
   - DB 연동 테스트 (테스트 DB 사용)
   - 인증/인가 흐름 테스트
   - 에러 응답 테스트

3. 테스트 환경 설정
   - 테스트용 설정 파일 생성
   - Mock/Stub 필요 시 생성
   - 테스트 데이터(Fixture) 생성

[출력]
- 테스트 코드 파일 목록
- 테스트 설정 파일
- 테스트 실행 명령어 안내
```

## 출력

| 산출물 | 형식 | 설명 |
|--------|------|------|
| 테스트 코드 파일들 | 각 프레임워크별 | 단위/통합 테스트 |
| 테스트 설정 파일 | 각 프레임워크별 | 테스트 환경 설정 |
| test_manifest.md | Markdown | 테스트 파일 목록 및 실행 안내 |

## 후속 연계

- SK_test_run: 생성된 테스트 실행
- Sol.4에서도 재사용 (수정 후 테스트 생성)
