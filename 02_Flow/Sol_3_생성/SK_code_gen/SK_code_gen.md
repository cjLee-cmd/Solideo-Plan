# SK_code_gen

## 목적

확정된 설계 문서(design.md)를 기반으로 소스코드를 생성한다. 전체 프로젝트 또는 특정 모듈만 선택적으로 생성할 수 있다.

## 입력 파라미터

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| design_file | Y | design.md 파일 경로 |
| scope | N | 생성 범위 (full/module). 기본값: full |
| target_module | N | scope=module일 때 대상 모듈명 |
| language | N | 대상 언어 (design.md에 명시된 경우 자동) |
| output_path | N | 출력 디렉토리 (기본: ./src) |

## 실행 프롬프트

```
당신은 행정망 소프트웨어 개발 전문가입니다.
설계 문서에 따라 소스코드를 생성하십시오.

[설계 문서]
{design_file 내용}

[생성 범위]
범위: {scope}
대상 모듈: {target_module}
대상 언어: {language}

[코드 생성 규칙]

1. 구조
   - design.md의 디렉토리 구조를 그대로 따른다
   - 계층별 패키지/모듈 분리를 준수한다
   - 설정 파일(application.yml, .env 등)을 포함한다

2. 코드 품질
   - 함수/메서드는 단일 책임 원칙을 따른다
   - 변수명/함수명은 설계 문서의 용어를 사용한다
   - 주요 로직에 한국어 주석을 포함한다
   - 에러 처리를 포함한다

3. 보안
   - 입력값 검증을 포함한다
   - SQL 파라미터 바인딩을 사용한다
   - 인증/인가 코드를 설계에 따라 구현한다
   - 감사 로그 코드를 포함한다

4. 폐쇄망 고려
   - 외부 네트워크 호출 코드를 포함하지 않는다
   - 모든 의존성은 내부 저장소에서 제공 가능한 것만 사용한다
   - CDN 참조 대신 로컬 리소스를 사용한다

[출력]
- 생성된 파일 목록 (경로, 설명)
- 디렉토리 구조 트리
- 빌드/실행 방법 안내
```

## 출력

| 산출물 | 형식 | 설명 |
|--------|------|------|
| 소스코드 파일들 | 각 언어별 | 디렉토리 구조 포함 생성 |
| file_manifest.md | Markdown | 생성된 파일 목록 및 설명 |
