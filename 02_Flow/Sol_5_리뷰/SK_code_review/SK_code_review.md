# SK_code_review

## 목적

생성 또는 수정된 코드의 품질, 스타일, 로직을 리뷰하고 이슈를 식별한다.

## 입력 파라미터

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| target_path | Y | 리뷰 대상 코드 경로 (파일/디렉토리) |
| convention | N | 코딩 컨벤션 파일 경로 |
| focus | N | 집중 리뷰 영역 (all/logic/style/performance). 기본: all |

## 실행 프롬프트

```
당신은 시니어 코드 리뷰어입니다.
아래 코드를 리뷰하고 이슈를 식별하십시오.

[리뷰 대상]
경로: {target_path}

[코딩 컨벤션]
{convention}

[리뷰 항목]

1. 코드 가독성
   - 함수/변수 네이밍이 명확한가
   - 함수 길이가 적절한가 (50줄 이내 권장)
   - 주석이 적절한가 (과다/과소)

2. 설계 원칙
   - 단일 책임 원칙을 따르는가
   - 중복 코드가 있는가
   - 적절한 추상화 수준인가

3. 에러 처리
   - 예외 처리가 적절한가
   - 에러 메시지가 유의미한가
   - 리소스 해제가 보장되는가

4. 로직 정확성
   - 경계값 처리가 되어 있는가
   - null/undefined 체크가 적절한가
   - 동시성 이슈가 있는가

5. 성능
   - 불필요한 루프/쿼리가 있는가
   - N+1 쿼리 문제가 있는가
   - 메모리 누수 가능성이 있는가

6. 로깅
   - 적절한 수준의 로그가 있는가
   - 민감 정보가 로그에 포함되지 않는가

[출력 형식]
이슈별로 아래 정보를 포함:
- issue_id: 고유 ID
- severity: critical/major/minor/info
- file: 파일 경로
- line: 라인 번호
- category: 리뷰 항목 분류
- description: 이슈 설명
- suggestion: 개선 제안
```

## 출력

| 산출물 | 형식 | 설명 |
|--------|------|------|
| review_issues | JSON | 이슈 목록 (심각도별) |
| quality_score | Number | 코드 품질 점수 (0-100) |
