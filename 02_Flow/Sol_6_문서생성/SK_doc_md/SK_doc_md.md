# SK_doc_md

## 목적

다양한 유형의 기술 문서를 마크다운 형식으로 생성한다. 각 문서 유형에 맞는 기본 템플릿을 제공한다.

## 입력 파라미터

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| doc_type | Y | 문서 유형 (design/analysis/test_report/minutes/proposal/manual/custom) |
| source_data | Y | 원본 데이터 (텍스트, JSON, 파일 경로) |
| template | N | 사용자 정의 템플릿 파일 경로 |
| output_path | N | 출력 경로 (기본: ./{doc_type}.md) |
| project_name | N | 프로젝트명 (문서 헤더용) |

## 실행 프롬프트

```
당신은 한국어 기술 문서 작성 전문가입니다.
아래 데이터를 기반으로 마크다운 문서를 작성하십시오.

[문서 유형]
{doc_type}

[원본 데이터]
{source_data}

[프로젝트명]
{project_name}

[문서 유형별 기본 구성]

### design (설계서)
1. 문서 정보 (제목, 작성일, 버전, 작성자)
2. 프로젝트 개요
3. 아키텍처 설계
4. 모듈 구조
5. API 설계
6. DB 스키마
7. 기술스택
8. 보안 설계
9. 변경 이력

### analysis (분석서)
1. 문서 정보
2. 분석 대상 개요
3. 기술스택 분석
4. 구조 분석
5. 의존성 분석
6. 코드 메트릭스
7. 개선 제안
8. 변경 이력

### test_report (테스트 보고서)
1. 문서 정보
2. 테스트 개요
3. 테스트 환경
4. 테스트 결과 요약
5. 상세 결과
6. 실패 분석
7. 커버리지
8. 결론

### minutes (회의록)
1. 회의 정보 (일시, 장소, 참석자)
2. 안건
3. 논의 내용
4. 결정 사항
5. 후속 조치 (담당자, 기한)

### proposal (제안서)
1. 문서 정보
2. 제안 배경
3. 현황 분석
4. 제안 내용
5. 기대 효과
6. 추진 일정
7. 소요 자원

### manual (사용자 매뉴얼)
1. 문서 정보
2. 개요
3. 설치/설정
4. 사용 방법
5. 주의사항
6. FAQ

[작성 규칙]
- 한국어로 작성
- 표와 목록을 적극 활용
- 기술 용어에 간단한 설명 병기
- 마크다운 문법 준수
```

## 출력

| 산출물 | 형식 | 설명 |
|--------|------|------|
| {doc_type}.md | Markdown | 생성된 문서 |
