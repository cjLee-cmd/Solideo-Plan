# SK_design_review 사용자 매뉴얼

## 개요

설계 문서(design.md)를 섹션별로 검토하고 승인/수정 피드백을 수집하는 Skill이다.

## opencode에서 사용하기

### 기본 호출

```
/skill SK_design_review --input design.md
```

### 사용자 정의 체크리스트 사용

```
/skill SK_design_review --input design.md --checklist my_checklist.md
```

## 검토 진행 흐름

1. 설계 문서의 각 섹션이 순서대로 표시된다
2. 각 섹션에 대해 검증 체크리스트 결과가 제시된다
3. 사용자가 각 섹션에 대해 승인/수정요청/의견을 입력한다
4. 모든 섹션 검토 후 종합 결과가 출력된다

## 검증 체크리스트 항목

- 기능 완전성 -- 요구사항 반영 여부
- 기술 타당성 -- 기술스택 적합성, 폐쇄망 호환성
- 보안 고려 -- 인증/인가, 민감데이터, 감사로그
- 성능 요구사항 -- 동시접속, 응답시간
- 확장성/유지보수성 -- 결합도, 확장 용이성

## 출력물

- `review_result.md` -- 섹션별 검토 결과 및 피드백 종합
- `design.md` (수정본) -- 피드백 반영 시 수정된 설계 문서

## 검토 결과 유형

| 결과 | 설명 | 후속 |
|------|------|------|
| 승인 | 설계 문서 확정 | Sol.3 생성 단계 진행 |
| 조건부승인 | 경미한 수정 후 진행 | 수정 반영 후 확정 |
| 반려 | 재설계 필요 | SK_design_spec으로 회귀 |

## 주의사항

- 반려 시 수정 사유를 구체적으로 입력해야 SK_design_spec이 정확히 반영할 수 있다
- 보안 관련 항목이 부적합이면 자동으로 반려 처리된다
