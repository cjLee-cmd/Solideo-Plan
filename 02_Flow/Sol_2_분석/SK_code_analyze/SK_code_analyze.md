# SK_code_analyze

## 목적

기존 소스코드 프로젝트를 분석하여 기술스택, 디렉토리 구조, 모듈 의존성, 코드 메트릭스를 도출한다.

## 입력 파라미터

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| project_path | Y | 분석 대상 프로젝트 디렉토리 경로 |
| scope | N | 분석 범위 (full/module/file). 기본값: full |
| target_module | N | scope=module일 때 대상 모듈명 |

## 실행 프롬프트

```
당신은 소프트웨어 분석 전문가입니다.
아래 프로젝트를 분석하여 구조화된 결과를 산출하십시오.

[대상 프로젝트]
경로: {project_path}
분석 범위: {scope}

[분석 항목]

1. 기술스택 식별
   - 프로그래밍 언어 (버전 포함)
   - 프레임워크/라이브러리
   - 빌드 도구
   - 데이터베이스
   - 기타 미들웨어

2. 디렉토리 구조 매핑
   - 최상위 디렉토리 구조 (트리 형식)
   - 주요 디렉토리별 역할 설명
   - 설정 파일 목록 및 용도

3. 모듈 의존성 분석
   - 내부 모듈 간 의존 관계
   - 외부 라이브러리 의존성
   - 순환 의존성 여부

4. 코드 메트릭스
   - 총 파일 수 / 코드 라인 수
   - 언어별 비율
   - 주요 클래스/함수 목록
   - 복잡도 높은 파일 식별

5. 폐쇄망 호환성 점검
   - 외부 네트워크 호출 코드 식별
   - 외부 CDN/API 의존 여부
   - 라이선스 확인 필요 항목

[출력 형식]
JSON 구조체로 각 분석 항목의 결과를 반환하십시오.
```

## 출력

| 산출물 | 형식 | 설명 |
|--------|------|------|
| analysis_result | JSON | 구조화된 분석 결과 |

## 후속 연계

- SK_analyze_doc: 분석 결과를 마크다운 문서로 변환
- SSA_dep_scanner: 의존성 상세 스캔 (서브태스크로 호출)
