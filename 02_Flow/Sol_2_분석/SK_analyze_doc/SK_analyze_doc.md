# SK_analyze_doc

## 목적

SK_code_analyze의 분석 결과를 받아 사람이 읽기 쉬운 마크다운 보고서로 변환한다.

## 입력 파라미터

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| analysis_result | Y | SK_code_analyze의 JSON 분석 결과 또는 파일 경로 |
| output_path | N | 출력 파일 경로 (기본: ./analysis_report.md) |
| template | N | 보고서 템플릿 (기본 템플릿 사용) |

## 실행 프롬프트

```
당신은 기술 문서 작성 전문가입니다.
아래 코드 분석 결과를 마크다운 보고서로 작성하십시오.

[분석 결과]
{analysis_result}

[보고서 구성]

# 코드 분석 보고서

## 1. 프로젝트 개요
- 프로젝트명, 분석일, 분석 범위
- 한 줄 요약

## 2. 기술스택
| 구분 | 항목 | 버전 | 비고 |
표 형식으로 정리

## 3. 디렉토리 구조
트리 형식으로 표시, 주요 디렉토리 역할 설명

## 4. 아키텍처 분석
- 계층 구조 설명
- 주요 모듈 역할
- 데이터 흐름

## 5. 의존성 분석
### 5.1 내부 모듈 의존성
모듈 간 관계를 표로 정리

### 5.2 외부 라이브러리
| 패키지명 | 버전 | 용도 | 라이선스 | 폐쇄망 호환 |
표 형식으로 정리

## 6. 코드 메트릭스
- 총 파일 수, 코드 라인 수
- 언어별 비율
- 복잡도 상위 파일 목록

## 7. 폐쇄망 호환성 점검
- 외부 네트워크 의존 항목
- 조치 필요 사항

## 8. 개선 제안
- 구조적 개선점
- 기술 부채 식별
- 우선순위별 권고사항

[작성 규칙]
- 기술 용어에 간단한 설명을 병기
- 표와 목록을 적극 활용
- 각 섹션에 핵심 요약을 먼저 배치
```
