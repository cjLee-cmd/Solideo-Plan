대상 코드를 읽고 아래 항목을 리뷰하라.

## 리뷰 항목

1. **코드 가독성** - 함수/변수 네이밍 명확성, 함수 길이(50줄 이내 권장), 주석 적절성
2. **설계 원칙** - 단일 책임 원칙, 중복 코드, 추상화 수준
3. **에러 처리** - 예외 처리 적절성, 에러 메시지 유의미성, 리소스 해제 보장
4. **로직 정확성** - 경계값 처리, null/undefined 체크, 동시성 이슈
5. **성능** - 불필요한 루프/쿼리, N+1 쿼리, 메모리 누수 가능성
6. **로깅** - 적절한 수준의 로그, 민감 정보 로그 포함 여부

## 이슈 보고 형식

각 이슈에 대해 아래 정보를 포함하라:
- **심각도**: critical(장애/보안, 즉시 수정) / major(품질 저하, 수정 권장) / minor(경미한 개선) / info(참고)
- **파일**: 파일 경로
- **라인**: 라인 번호
- **설명**: 이슈 내용
- **제안**: 개선 방안

코드 품질 점수(0-100)를 산출하라.
