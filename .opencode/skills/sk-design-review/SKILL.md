design.md를 읽고 아래 체크리스트에 따라 섹션별로 검증하라. 각 섹션의 검증 결과를 사용자에게 제시하고 승인/수정/반려 피드백을 수집하라.

## 검증 체크리스트

1. **기능 완전성** - 모든 요구사항이 설계에 반영되었는가, 누락된 기능이 있는가
2. **기술 타당성** - 기술스택이 요구사항을 충족하는가, 폐쇄망에서 모든 구성요소가 동작 가능한가
3. **보안 고려** - 인증/인가 방식이 행정망 기준을 충족하는가, 민감 데이터 처리가 적절한가, 감사 로그가 포함되었는가
4. **성능 요구사항** - 동시 사용자 수 기준이 명시되었는가, 응답시간 목표가 현실적인가
5. **확장성/유지보수성** - 모듈 간 결합도가 낮은가, 향후 기능 추가가 용이한 구조인가

## 검토 진행 방식

1. 각 섹션을 순서대로 사용자에게 요약 제시
2. 섹션별 검증 항목 결과 표시 (적합/부적합/확인필요)
3. 사용자에게 승인/수정요청/의견 입력 요청
4. 모든 섹션 검토 완료 후 종합 결과 출력

## 출력

- review_result.md에 섹션별 검토 결과, 수정 요청 사항, 최종 승인 상태(승인/조건부승인/반려) 기록
- 반려 시 수정 사유를 구체적으로 기록하라
- 보안 관련 항목이 부적합이면 자동 반려 처리하라
