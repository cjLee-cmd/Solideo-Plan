사용자로부터 아래 정보를 수집하여 design.md 설계 문서를 작성하라.

## 수집 항목

- 프로젝트명
- 프로젝트 목적 및 배경
- 기능 요구사항 목록
- 기술스택 제약조건 (미지정 시 Java/Spring 기본)
- 배포 환경 (기본: 폐쇄망)
- 연동 대상 기존 시스템

## design.md 작성 구성

1. **프로젝트 개요** - 프로젝트명, 목적, 범위, 용어 정의
2. **아키텍처 설계** - 시스템 구성도, 계층 구조(Presentation/Business/Data), 통신 방식
3. **모듈 구조** - 모듈 목록/역할, 모듈 간 의존성, 디렉토리 구조
4. **API 설계** - 엔드포인트 목록, 요청/응답 스키마, 인증/인가 방식
5. **DB 스키마** - 테이블/컬럼 정의, 관계 정의, 인덱스 전략
6. **기술스택 결정** - 언어/프레임워크/라이브러리, 선정 사유, 폐쇄망 호환성 확인
7. **보안 설계** - 인증/인가, 데이터 암호화, 감사 로그 정책
8. **비기능 요구사항** - 성능 목표, 가용성, 확장성

## 필수 규칙

- 폐쇄망 환경을 전제한다. 외부 네트워크 의존 요소를 배제하라.
- 행정망 보안 지침을 준수하라. 인증/인가, 암호화, 감사로그는 필수 포함이다.
- 모든 외부 라이브러리는 폐쇄망 내부 저장소에서 제공 가능한 것만 선정하라.
- 마크다운 형식으로 작성하라.
- 완성된 문서를 design.md로 저장하라.
