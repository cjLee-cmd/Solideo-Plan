대상 코드를 읽고 아래 기준으로 보안 취약점을 점검하라.

## OWASP Top 10 점검

- A01 접근제어 결함 - 인가 우회, 권한 상승
- A02 암호화 실패 - 평문 저장, 약한 암호화
- A03 주입 공격 - SQL Injection, Command Injection
- A04 불안전한 설계 - 비즈니스 로직 결함
- A05 보안 구성 오류 - 디버그 모드, 기본 계정
- A06 취약한 컴포넌트 - 알려진 취약점 라이브러리
- A07 인증 실패 - 세션 관리, 비밀번호 정책
- A08 데이터 무결성 실패 - 검증 없는 역직렬화
- A09 로깅/모니터링 실패 - 감사로그 부재
- A10 SSRF - 서버 측 요청 위조

## 행정망 추가 점검

- 개인정보보호법 준수 (수집/처리/저장/파기)
- 접근제어 정책 (역할 기반, 최소 권한)
- 감사로그 (접근/변경/삭제 기록)
- 데이터 암호화 (전송/저장)
- 세션 관리 (타임아웃, 동시 세션 제한)
- 입력값 서버 측 검증, 파일 업로드 검증

## 취약점 보고 형식

- **심각도**: critical/high/medium/low
- **OWASP 분류**: A01~A10
- **파일/라인**: 위치
- **위험 시나리오**: 공격 가능 경로
- **수정 가이드**: 구체적 수정 방법

critical/high 취약점은 수정 전까지 배포 금지로 판정하라.
