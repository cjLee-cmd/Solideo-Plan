change_summary.md, 영향분석 결과, 테스트 결과를 읽고 종합 보고서를 작성하라.

## 보고서 구성

1. **수정 요약** - 수정일, 수정 유형(bugfix/feature/refactor), 요구사항 요약
2. **변경 파일 목록** - 표(파일 경로, 변경 유형, 변경 라인 수, 설명)
3. **변경 상세** - 파일별 주요 변경 내용 Diff 발췌
4. **영향분석 결과** - 표(영향 유형, 대상, 위험도, 조치 상태)
5. **테스트 결과** - 실행 결과 요약, 실패 항목 및 조치
6. **검증 상태** - verify.md 기준 대비 검증 통과 여부 (있는 경우)
7. **잔여 리스크** - 미검증 항목, 추가 모니터링 필요 사항

modification_report.md로 저장하라.
