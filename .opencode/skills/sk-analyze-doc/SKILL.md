analysis_result.json을 읽고 아래 구성의 마크다운 보고서를 작성하라.

## 보고서 구성

1. **프로젝트 개요** - 프로젝트명, 분석일, 분석 범위, 한 줄 요약
2. **기술스택** - 표 형식(구분, 항목, 버전, 비고)
3. **디렉토리 구조** - 트리 형식, 주요 디렉토리 역할 설명
4. **아키텍처 분석** - 계층 구조, 주요 모듈 역할, 데이터 흐름
5. **의존성 분석** - 내부 모듈 관계 표, 외부 라이브러리 표(패키지명, 버전, 용도, 라이선스, 폐쇄망 호환)
6. **코드 메트릭스** - 파일 수, 라인 수, 언어별 비율, 복잡도 상위 파일
7. **폐쇄망 호환성 점검** - 외부 네트워크 의존 항목, 조치 필요 사항
8. **개선 제안** - 구조적 개선점, 기술 부채, 우선순위별 권고사항

## 작성 규칙

- 한국어로 작성하라
- 표와 목록을 적극 활용하라
- 각 섹션에 핵심 요약을 먼저 배치하라
- analysis_report.md로 저장하라
