## 2. 테스트 개요

### 2.1 테스트 목적

본 보고서는 행정안전망 NTIS(국가과학기술지식정보서비스) 신규사업 관리 모듈(SNTFMW Meal 모듈)에 대해 수행한 코드 리뷰 및 보안 취약점 점검 결과를 종합하여, 시스템의 프로덕션 배포 적합성을 평가한다.

### 2.2 테스트 대상

| **모듈명** | SNTFMW Meal 모듈 (EJB + Struts Action) |
| **대상 파일** | 5개 Java 소스 파일 |
| **기술 스택** | EJB 2.x, Struts 1.x, Java |

## 5. 상세 테스트 결과

### 5.1 Critical 이슈 (즉시 수정 필요)

| 이슈 ID | 제목 | 영향 파일 | 심각도 | 상태 |
|---------|------|-----------|--------|------|
| REV-001 | DAO에서 QueryService가 null 상태로 초기화되지 않음 | SntFmwMealEJBDAO.java | Critical | 미해결 |
| REV-002 | Action에서 빈 문자열을 request attribute 키로 사용 | SntFmwMealAction.java | Critical | 미해결 |
| REV-003 | 필수 import 문 누락 | SntFmwMealAction.java, SntFmwMealLocal.java | Critical | 미해결 |
| REV-004 | 인가 검증 전무 (Broken Access Control) | SntFmwMealAction.java 전역 | Critical | 미해결 |
| REV-005 | SQL Injection 취약점 가능성 | SntFmwMealEJBDAO.java | Critical | 미해결 |

#### REV-001: DAO에서 QueryService가 null 상태로 초기화되지 않음

- **테스트 항목**: DAO 초기화 및 의존성 주입
- **테스트 방법**: SntFmwMealEJBDAO.java 소스 코드 분석
- **테스트 결과**: ❌ 실패
- **상세 내용**: `queryservice` 필드가 `null`로 선언되고 초기화되지 않음. 모든 메서드에서 NullPointerException 발생
- **재현 경로**: `SntFmwMealEJBDAO.java:28` — 필드 선언 시 초기화 코드 부재
- **권장 조치**: 생성자 또는 DI를 통해 QueryService 인스턴스 초기화

#### REV-002: Action에서 빈 문자열을 request attribute 키로 사용

- **테스트 항목**: Request Attribute 전달 검증
- **테스트 방법**: SntFmwMealAction.java 소스 코드 분석
- **테스트 결과**: ❌ 실패
- **상세 내용**: `request.setAttribute("", invokeLocal(request))` — 빈 키로 인해 뷰에 데이터 전달 불가
- **재현 경로**: `SntFmwMealAction.java:35,42,49`
- **권장 조치**: 의미 있는 키 이름으로 변경 (예: `"result"`)

#### REV-003: 필수 import 문 누락

- **테스트 항목**: 컴파일 가능성 검증
- **테스트 방법**: import 문 분석
- **테스트 결과**: ❌ 실패
- **상세 내용**: `HashMap`, `Collection`, `ArrayList`, `Log`, `NTIS_COMMON_DEFINE`, `DefaultParameters` import 누락
- **재현 경로**: `SntFmwMealAction.java:18-27`, `SntFmwMealLocal.java:8`
- **권장 조치**: 누락된 import문 추가

#### REV-004: 인가 검증 전무 (Broken Access Control)

- **테스트 항목**: 접근 제어 검증
- **테스트 기준**: OWASP A01:2021
- **테스트 방법**: 액션 메서드별 권한 검증 로직 확인
- **테스트 결과**: ❌ 실패
- **상세 내용**: 모든 CRUD 액션 메서드에서 권한 검증 없이 `invokeLocal()` 호출. 인증된 사용자라면 누구나 타인의 데이터 조회/수정/삭제 가능
- **재현 경로**: `SntFmwMealAction.java:17-51` 전 메서드
- **권장 조치**: 각 액션 메서드 시작 부분에 인증 확인 + 역할(Role) 기반 인가 검증 추가

#### REV-005: SQL Injection 취약점 가능성

- **테스트 항목**: SQL Injection 방어 검증
- **테스트 기준**: OWASP A03:2021
- **테스트 방법**: QueryService 호출부 파라미터 전달 방식 분석
- **테스트 결과**: ❌ 실패
- **상세 내용**: `QueryService.find()` / `update()`에 전달되는 파라미터가 문자열 배열로 직접 전달. `cud_data` 파싱 데이터가 사용자 입력인 경우 SQL Injection 가능
- **재현 경로**: `SntFmwMealEJBDAO.java:28-29, 53, 72, 80, 98-100`
- **권장 조치**: PreparedStatement 파라미터 바인딩 확인, `smode` 값 화이트리스트 검증 추가

#### REV-006: 개인정보 암호화 취약점

- **테스트 기준**: OWASP A02:2021
- **테스트 결과**: ❌ 실패
- **상세 내용**: 커스텀 `CCDEncryption` 사용 — 표준 알고리즘 여부 불명. 복호화된 데이터가 HashMap에 담겨 메모리 상 평문 노출. 암호화 키 관리 방식 불명

#### REV-007: 감사로그 부재

- **테스트 기준**: OWASP A09:2021
- **테스트 결과**: ❌ 실패
- **상세 내용**: 에러 로깅이 `"NPE"`, `"ERR"`만 기록. CRUD 작업에 대한 감사로그(누가, 언제, 무엇을) 전혀 없음. 개인정보(SSN) 접근 이력 추적 불가 — 개인정보보호법 위반 소지
- **권장 조치**: 감사로그 AOP 적용, 개인정보 접근로그 기록(조회자ID, 시간, 대상, 사유), 접근이력 1년 이상 보관

#### REV-008: 불안전한 비즈니스 로직 설계

- **테스트 기준**: OWASP A04:2021
- **테스트 결과**: ❌ 실패
- **상세 내용**: `cud_data` 문자열을 직접 파싱하여 DB 작업. 트랜잭션 경계 불명확 — 일부 성공 후 실패 시 데이터 불일치 가능
- **권장 조치**: JSON 등 구조화된 포맷으로 변경, 작업 건수 제한, 명시적 트랜잭션 경계 설정

### 7.1 실패 원인 분류

| 원인 유형 | 건수 | 비율 | 설명 |

### 7.2 근본 원인 분석 (Root Cause Analysis)

1. **개발 표준 미준수**: Java 코딩 컨벤션, 보안 코딩 가이드라인이 개발 과정에 반영되지 않음
2. **레거시 프레임워크 의존**: EJB 2.x + Struts 1.x는 현대 보안 표준을 지원하지 않음
3. **보안 검토 프로세스 부재**: 개발 완료 후 보안 검토가 수행되지 않은 것으로 판단됨
4. **테스트 코드 부재**: 단위 테스트, 통합 테스트 코드가 존재하지 않음
